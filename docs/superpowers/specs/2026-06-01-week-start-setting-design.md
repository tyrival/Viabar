# 每周开始日设置设计说明

日期：2026-06-01

## 目标

在 `设置 > 显示 > 视图` 中增加 `每周开始于` 配置，允许用户选择周日或周一。总览右侧汇总面板中的周待办和周完成区间统一按该设置计算，不再直接依赖系统日历的动态周起始日。

## 已确认的产品决策

- 新增设置项 `每周开始于`，提供 `周日`、`周一` 两个选项。
- 设置项位于 `设置 > 显示 > 视图` 分组中，排列在 `总览` 与 `日期格式` 之间。
- 全新安装首次创建设置记录时，根据当前系统地区初始化：
  - 美国地区默认 `周日`；
  - 其他地区默认 `周一`。
- 已安装用户升级后首次获得该字段时，也根据升级时的当前系统地区初始化一次。
- 用户修改后保存明确值，不再随系统地区变化。
- 总览面板现有选项保持不变：
  - `周待办` 可选 `本周` / `下周`；
  - `周完成` 可选 `本周` / `上周`。
- 月完成区间不受影响。
- 未获另行授权时不编译代码，仅执行源码检查、字符串资源检查与 `git diff --check`。

## 方案

采用持久化明确值方案：

- 新增 `WeekStartDay` 枚举，保存 `sunday` 或 `monday`。
- `AppSettings` 新增 `weekStartDay` 持久化字段。
- `WeekStartDay` 负责：
  - 根据地区返回首次初始化默认值；
  - 从存储值解析出有效枚举；
  - 对无效值回退到当前地区默认值；
  - 为总览构建器生成固定周起始日的日历。
- `AppSettingsStore.ensureDefaultSettings(in:)` 在启动补齐设置记录时，将缺失或无效值立即写成当前地区默认值并保存，确保地区只参与一次初始化。
- `OverviewReportBuilder` 继续保持纯数据边界，新增周起始日输入，并用固定周起始日计算自然周区间。
- `ContentView` 从单例 `AppSettings` 读取保存值，解析后传给 `OverviewReportBuilder`。

不增加 `系统` 选项。系统地区只用于字段首次初始化或旧数据兼容回退；一旦设置中存在有效的 `sunday` / `monday`，总览周区间始终按保存值计算。

## 默认值与升级兼容

### 地区判断

首次初始化接收当前系统 `Locale`：

- `Locale.region` 为美国地区时返回 `sunday`；
- 其他地区或无法识别地区时返回 `monday`。

地区判断只区分美国与非美国，不根据界面语言推断。

### SwiftData 兼容

`AppSettings.weekStartDay` 使用可兼容现有持久化记录的可选字段定义。新记录直接写入当前地区默认值；既有记录由 `AppSettingsStore.ensureDefaultSettings(in:)` 在启动时检查，缺失或无效时立即写入当前地区默认值并保存。解析层仍保留地区回退，作为异常数据的防御性处理。

### 备份兼容

- `BackupSettingsSnapshot` 保存 `weekStartDay`。
- 新备份恢复时写回明确保存值。
- 旧备份缺少该键时，解码后使用恢复时系统地区默认值，并将明确值写入设置记录。
- 无效备份值按恢复时系统地区默认值回退，并将明确值写入设置记录。

## 总览区间规则

`OverviewReportBuilder` 的月区间算法保持不变。周区间统一从传入的 `WeekStartDay` 派生：

- `sunday`：自然周为周日 `00:00` 至下一周周日 `00:00` 的半开区间；
- `monday`：自然周为周一 `00:00` 至下一周周一 `00:00` 的半开区间。

区间继续使用 `start <= timestamp < end`。

### 周完成

- `本周` 使用当前自然周；
- `上周` 使用紧邻当前自然周之前的完整自然周；
- 周起始日切换后，已完成任务与子任务按照新边界重新归组。

### 周待办

- `本周` 使用当前自然周结束时间作为累计待办截止边界；
- `下周` 使用下一自然周结束时间作为累计待办截止边界；
- 既有“截至所选周末的累计未完成提醒”语义保持不变；
- 周起始日切换后，截止周末随设置变化。

## 设置界面

在 `SettingsView` 的 `显示 > 视图` 分组中增加：

```text
总览          [所有项目 / 星标项目]
每周开始于    [周日 / 周一]
日期格式      [格式示例]
```

Picker 沿用现有小尺寸控件与右对齐布局，不增加说明文案或额外交互。

## 本地化

在中英文字符串资源中增加：

- `每周开始于` / `Week Starts On`
- `周日` / `Sunday`
- `周一` / `Monday`

## 代码职责边界

- `Viabar/Models/AppSettings.swift`
  - 定义 `WeekStartDay`；
  - 保存并解析 `weekStartDay`；
  - 提供地区默认值与固定周起始日日历。
  - 启动时将缺失或无效值补齐为明确值并保存。
- `Viabar/Models/OverviewReport.swift`
  - 接收周起始日并统一计算周待办、周完成区间。
- `Viabar/ContentView.swift`
  - 读取当前设置并传递给汇总构建器。
- `Viabar/Views/Settings/SettingsView.swift`
  - 在显示页视图分组中增加 Picker。
- `Viabar/Models/BackupSnapshot.swift`
  - 在备份快照中保存新字段，并保持旧备份可解码。
- `Viabar/Services/BackupService.swift`
  - 恢复备份时写回解析后的明确值。
- `Viabar/en.lproj/Localizable.strings`
  - 增加英文文本。
- `Viabar/zh-Hans.lproj/Localizable.strings`
  - 增加简体中文文本。
- `ViabarTests/ViabarTests.swift`
  - 增加默认值、解析、周边界和备份兼容测试源码。

## 静态验证策略

- 核对美国地区默认周日，其他地区默认周一。
- 核对有效保存值不会随地区变化。
- 核对无效或缺失保存值回退到当前地区默认值。
- 核对同一参考时间在周日和周一配置下得到不同的自然周边界。
- 核对 `周完成 > 本周 / 上周` 和 `周待办 > 本周 / 下周` 都使用新设置。
- 核对新备份保存并恢复明确值，旧备份缺少字段时仍可解码并按当前地区回退。
- 核对两份字符串资源均包含新增文本。
- 运行 `git diff --check`。
