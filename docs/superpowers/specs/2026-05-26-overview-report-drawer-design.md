# 总览汇总侧边栏设计说明

日期：2026-05-26

## 目标

在 Viabar 总览页面增加一个右侧汇总面板，集中呈现近期已完成内容与下周需要处理的提醒内容。面板沿用项目详情中备忘录侧边栏的抽屉交互与面板配色，并用只读项目卡片承载任务树，方便用户阅读和整组复制。

本设计延续现有能力：

- `ContentView` 已负责主面板右上角控件及项目详情备忘录抽屉的展示协调；
- `Milestone.completedAt` 与 `SubTask.completedAt` 已保存完成时间；
- `NotificationScheduleEntry` 是当前有效提醒的时间轴，归档项目会从时间轴移除但保留提醒配置；
- `ViabarColor.mainPanelMemoBackground` 与 `MemoTimelineStyle` 已形成备忘录面板的浅色/深色视觉语义；
- `MenuBarProjectCardView` 已形成按项目组织任务内容的卡片视觉参考；
- `AppLocalization` 与现有本地化资源负责显示文本的语言切换。

## 已确认的产品决策

- 右侧汇总面板仅属于总览页面，不替代项目详情的备忘录面板。
- 总览面板与备忘录面板具有相同宽度、背景、分隔线、展开/收起动画和右上角切换按钮交互。
- 展开/收起按钮位于面板自身右上角；面板不显示“工作汇总”或其他总标题。
- 面板从上到下固定为 `本周完成`、`下周待办`、`本月完成` 三个分组，并以横向分隔线分区。
- 面板读取全部项目；归档项目可以显示在完成历史中，但不显示在下周待办中。
- 本周与本月的完成内容按任务/子任务实际完成时间筛选，而不是按项目状态或提醒状态推测。
- 下周待办读取下一个自然周内仍处于有效提醒时间轴上的条目。
- 项目级提醒指向第一个未完成父任务；当该父任务包含未完成子任务时，显示父任务及其下方缩进的全部未完成子任务。
- 同一分组、同一项目内，同一个任务或子任务即使被多个提醒来源命中，也只展示和复制一次。
- 卡片及复制交互均为只读；不在该面板中提供任务完成复选框或编辑动作。
- 新增界面文本须适配现有国际化流程，面板与卡片颜色须适配浅色和深色模式。
- 本轮实现未获另行授权时不以编译代码排查问题，验证默认止于静态检查与 `git diff --check`。

## 实现方案

采用独立的总览汇总构建器与专属右栏视图：

- `ContentView` 管理总览面板的显隐、右侧占位和按钮层级；
- 新增纯数据构建器，将全部项目、有效提醒时间轴条目、参考日期与日历转换为三个汇总分组；
- 新增只读右栏视图，按照构建器输出渲染分组标题、复制动作和项目卡片；
- 卡片外观借鉴菜单栏卡片，面板框架和反馈交互借鉴备忘录面板；
- 复制文本由构建器或相邻格式化器集中生成，不在多个视图中重复拼接规则。

不扩展 `MenuBarContentBuilder` 来承载总览汇总规则。菜单栏针对当天行动、可勾选完成和项目筛选；总览面板针对自然周/月报告、归档历史和只读复制，两者数据语义不同，强行共用会混合行为边界。

## 面板布局与交互

### 展示边界

- 当前选择为总览时，主内容可在右侧展示汇总面板。
- 当前选择为某个项目时，继续按既有规则展示该项目的备忘录面板，不展示总览汇总面板。
- 总览汇总面板维护自己的展开状态，不修改项目详情备忘录的展开状态。
- 总览面板初始状态与备忘录当前体验一致：首次进入可见，用户可通过面板右上角按钮收起或再次展开。

### 面板框架

- 展开宽度沿用备忘录抽屉的 `360pt` 规格。
- 展开时，面板占用总览内容的右侧空间，避免覆盖项目卡片。
- 背景使用备忘录面板相同的面板背景语义色；边界使用相同层级的竖向分隔线。
- 顶部仅为按钮容纳区域，不显示标题或搜索框。
- 展开/收起按钮的圆形背景、阴影、图标尺寸、hover 状态及帮助文本行为与备忘录切换按钮一致，显示文本须本地化。

### 三个分组

- 三个分组顺序固定为 `本周完成`、`下周待办`、`本月完成`。
- 每组顶部为紧凑标题行，标题使用次级灰色文本与较小行高。
- 相邻分组使用横向分隔线分开。
- 每组标题行最右侧放置复制按钮；按钮仅显示 `doc.on.doc` 图标，无胶囊或方形背景。
- 复制按钮 hover 时显示手形指针并切换为蓝色强调；复制成功后以备忘录卡片相同方式短暂显示 `已复制` 反馈。
- 空分组保留复制按钮占位但禁用交互，使三组标题对齐稳定；分组内容显示简洁的本地化空状态。

## 日期范围与数据来源

### 自然周与自然月

汇总构建器接受 `now` 与 `Calendar`，便于稳定测试并遵从用户当前地区的自然周规则：

- `本周完成`：使用 `calendar.dateInterval(of: .weekOfYear, for: now)` 所表示的当前自然周区间。
- `本月完成`：使用 `calendar.dateInterval(of: .month, for: now)` 所表示的当前自然月区间。
- `下周待办`：从当前自然周结束时间开始，取下一段自然周区间。
- 所有区间均采用半开区间语义：`start <= timestamp < end`，避免周/月边界重复显示。

### 完成数据来源

- 已完成内容从项目任务树上的 `Milestone.completedAt` 与 `SubTask.completedAt` 读取。
- 完成分组遍历全部项目，包括 `isArchived == true` 的项目。
- 没有子任务的父任务：当自身 `completedAt` 位于区间内时，生成一条任务行。
- 包含子任务的父任务：父任务只作为层级上下文标题，不因其联动完成时间单独计为成果；只展示 `completedAt` 位于区间内的子任务。
- 包含子任务的父任务若在该区间内没有任何命中的子任务，则不会仅凭父任务 `completedAt` 生成空标题行。
- 通过现有“完成父任务”操作联动完成全部子任务时，子任务获得同一完成时间，因此会在该周期内完整显示在父任务下方。

### 下周待办数据来源

- 下周待办从 `NotificationScheduleEntry` 中筛选 `fireDate` 落入下一自然周的有效时间轴条目。
- 归档会移除项目的时间轴条目，因此归档项目自然不会进入下周待办；不使用归档项目保留的 `Reminder` 配置重新构造计划。
- 构建器通过条目的 `projectId` 与 `ownerKind` 回到实际项目及任务树，确保展示当前标题、当前未完成状态和当前父子结构。
- 已完成或已归档目标即使仍存在异常残留条目，也在构建结果中排除，避免显示不可执行内容。

## 任务树组装规则

### 完成分组

每个命中项目生成一张项目卡片。卡片内部按照项目当前任务顺序显示：

- 无子任务父任务显示为普通任务行。
- 有子任务父任务显示为父任务行，并在其下方缩进显示本周期命中的子任务。
- 不在完成卡片内展示完成时间；完成时间仅用于筛选。

### 下周待办：任务与子任务提醒

- `ownerKind == "milestone"` 的条目对应父任务本身，显示该父任务行；即使该父任务含有子任务，也不会因任务自身提醒而自动展开所有子任务。
- `ownerKind == "subtask"` 的条目显示其父任务上下文行，并在下方缩进显示命中的子任务。
- 若同一父任务下存在多个各自命中的子任务，仅显示一个父任务上下文行及多条缩进子任务行。

### 下周待办：项目级提醒

- `ownerKind == "project"` 的条目解析项目按 `orderIndex` 排列的第一条未完成父任务。
- 若该父任务没有未完成子任务，显示父任务本身作为一条待办。
- 若该父任务含有一个或多个未完成子任务，显示父任务作为上下文行，并在下方按 `orderIndex` 缩进显示全部未完成子任务；父任务不额外计作一条可执行子项。
- 该规则用于右栏阅读和复制，不改变既有系统通知文案目前只指向下一步任务标题的行为。

### 去重与顺序

- 去重范围为单一分组内的单一项目卡片。
- 父任务以 `milestoneId` 标识，子任务以 `taskId` 标识；多个提醒来源命中相同标识时只保留一次展示。
- 当项目级提醒已展开某个父任务的子任务时，命中其中同一子任务的独立提醒不会再生成重复行。
- 项目卡片按 `Project.orderIndex` 升序排列，名称作为相同顺序值时的稳定次级排序。
- 卡片内部父任务与子任务按各自 `orderIndex` 升序排列，使完成与待办分组均遵循项目中的既有结构。

## 卡片视觉设计

- 卡片使用菜单栏下拉面板项目卡片的圆角、底色、描边与纵向间距语义，并根据右栏宽度收紧内边距。
- 卡片不显示复选框、提醒时间、完成时间或提醒来源标签。
- 项目标题左侧显示项目 SF Symbol，颜色为 `Color(hex: project.accentColor)`。
- 项目名称颜色统一为 `ViabarColor.primary`，不跟随项目主题色。
- 归档项目在完成分组卡片右上方显示本地化的 `已归档` 次级灰色标签；活跃项目不显示该标签。
- 父任务、子任务文本均支持多行显示，不使用单行截断；子任务通过固定缩进形成清晰的父子层级。
- 浅色和深色面板底色、卡片底色、描边、标题灰度及复制 hover 蓝色遵循现有备忘录面板与系统语义色，不创建仅适用于单一外观的硬编码样式。

## 复制规则

### 操作范围

- 每个分组复制按钮仅复制该分组当前展示的全部项目卡片内容。
- 复制内容不包含完成时间、通知时间、归档标签、提醒来源、空状态或复制反馈文字。
- 空分组复制按钮为禁用状态，不修改剪贴板。

### 文本格式

- 项目前使用从 `1.` 开始的流水序号，并按卡片显示顺序递增。
- 父任务或无子任务的任务行前使用 `- `。
- 子任务行前先添加两个空格缩进，再使用 `- `。
- 相邻项目块之间保留一个空行。
- 若父任务仅作为子任务上下文显示，父任务仍以 `- ` 写入，随后写入缩进子任务。

示例：

```text
1. 网站改版
- 设计系统迁移
  - 整理组件色板
  - 完成按钮状态适配和深色模式修订

2. 客户访谈
- 访谈纪要归纳
```

## 本地化与空状态

新增或复用以下本地化文本，并通过当前 `AppLocalization` / 字符串资源路径提供语言版本：

- `本周完成`
- `下周待办`
- `本月完成`
- `已归档`
- `复制本周完成`、`复制下周待办`、`复制本月完成`
- `已复制`
- 三个分组各自的无内容提示
- 总览汇总面板的展开/收起帮助文本

不在构建器中嵌入界面语言文本；构建器输出语义类型和项目/任务内容，视图或复制格式化边界负责注入本地化 UI 文案。复制正文中的项目与任务标题沿用用户输入的原始内容，不翻译。

## 代码职责边界

- `Viabar/ContentView.swift`
  - 管理总览汇总面板显隐状态、右侧空间预留及总览专属切换按钮；
  - 保持项目详情备忘录抽屉逻辑不变，并确保两个页面各自只渲染相应右栏。
- `Viabar/Models/OverviewReport.swift`（新增）
  - 定义分组、项目卡片与层级任务行的只读展示模型；
  - 实现自然周/月范围计算、完成内容提取、下周时间轴映射、项目级提醒展开、去重与复制正文生成。
- `Viabar/Views/MainPanel/OverviewReportDrawerView.swift`（新增）
  - 渲染面板、分组、复制反馈和只读项目卡片；
  - 复用备忘录面板/菜单栏卡片的视觉语义，但不携带写操作。
- `Viabar/System/AppLanguageController.swift`、`Viabar/zh-Hans.lproj/Localizable.strings` 与 `Viabar/en.lproj/Localizable.strings`
  - 为新增标题、标签、帮助文本和空状态补充本地化文案访问路径。
- `ViabarTests/ViabarTests.swift`
  - 覆盖构建器和复制正文的确定性数据行为。

如果实现过程中发现备忘录与总览需要共用小范围面板样式常量，可抽取只涉及颜色或尺寸的轻量共享样式；不为本次功能重构备忘录的数据或交互结构。

## 验证策略

### 单元测试范围

为汇总构建器增加测试，至少覆盖：

- 当前自然周与当前自然月的起止边界，边界时间不会跨分组重复计入；
- 无子任务父任务按自身 `completedAt` 进入完成分组；
- 有子任务父任务只作为上下文，且仅区间内完成的子任务进入卡片；
- 归档项目可以进入本周/本月完成，但不会通过保留的提醒配置进入下周待办；
- 下周内的父任务提醒与子任务提醒映射为正确层级；
- 项目级提醒展开第一条未完成父任务下的全部未完成子任务；
- 同一任务由项目级提醒和自身提醒同时命中时仅显示一次；
- 项目排序、任务排序和复制正文格式稳定；
- 空分组无复制正文。

### 静态与视觉核验

- 核对总览面板按钮位于右栏内部右上角，且没有额外面板标题。
- 核对分组标题的紧凑灰色样式、横向分隔线、纯图标复制按钮与 hover 反馈。
- 核对项目图标项目色、项目名称 primary 蓝色、仅归档完成卡片展示 `已归档`。
- 核对长父任务与子任务文本多行展示及子任务缩进。
- 核对新增字符串接入现有国际化资源，并使用现有面板语义色支持浅色/深色模式。
- 实现完成后默认运行 `git diff --check` 检查文本与补丁质量；除非用户明确提出，不运行编译或构建命令来排查本功能。
