# 总览汇总侧边栏设计说明

日期：2026-05-26

## 目标

在 Viabar 总览页面增加一个右侧汇总面板，以可切换的周/月范围展示待办与完成内容。面板沿用项目详情备忘录侧边栏的抽屉交互与配色，并以只读项目卡片展示可跳转的任务树和提醒状态，便于回顾、安排和整组复制。

## 现有基础与已定位问题

- `ContentView` 管理顶部工具栏、页面选择、搜索跳转请求与项目详情备忘录抽屉。
- `Milestone.completedAt` 与 `SubTask.completedAt` 已保存实际完成时间。
- `Reminder` 保存提醒配置；`NotificationScheduleEntry` 仅表示仍需投递的当前时间轴条目。
- 任务/子任务单次提醒触发后会删除时间轴条目，但仍保留 `Reminder` 配置和原触发时间，可呈现为逾期待办。
- 项目级单次提醒当前在触发后会清除 `project.reminder`，无法在逾期面板中继续显示；本设计将修正这一生命周期不一致。
- `GlobalSearchNavigationRequest`、`MilestoneListView` 与高亮 modifier 已提供项目、任务、子任务跳转后的橙色反馈能力。
- 当前汇总抽屉按钮失效的根因是点击层级：按钮位于右栏顶部，而 `ContentView` 的顶层工具栏覆盖同一区域并拥有更高层级。修复应调整右栏/按钮的命中层级，不改变已确认的按钮位置。

## 已确认的产品决策

- 右侧汇总面板仅属于总览页面；项目详情继续显示既有备忘录面板。
- 展开时右栏宽度、背景、分隔线和抽屉动画等同于备忘录面板；展开/收起按钮位于右栏内部右上角，不显示总标题。
- 面板从上到下固定为 `周待办`、`周完成`、`月完成` 三个分组，并以横向分隔线分区。
- 每个分组标题后紧跟一个小型胶囊下拉选择器：
  - `周待办` 可选 `本周` / `下周`，默认 `本周`；
  - `周完成` 可选 `本周` / `上周`，默认 `本周`；
  - `月完成` 可选 `本月` / `上月`，默认 `上月`。
- 待办范围采用截至所选周末的累计语义：切到 `本周` 显示截至本周末仍未完成的全部有效提醒；切到 `下周` 显示截至下周末仍未完成的全部有效提醒，包含本周遗留未完成项。
- 面板只展示归档项目的完成历史；归档项目不出现在周待办中。
- 项目、任务与子任务的提醒在一张项目卡片内合并展示并按实体去重。
- 项目名称、任务、子任务都可点击跳转；任务与子任务跳转后复用搜索跳转的橙色高亮效果。
- 卡片和复制交互为只读；不提供复选框完成功能。
- 界面文本必须接入现有国际化资源，颜色必须适配浅色和深色模式。
- 未获另行授权时不编译代码排查或验证，只执行静态检查与 `git diff --check`。

## 实现方案

采用独立的总览汇总构建器、专属右栏视图和既有导航请求复用方案：

- `ContentView` 管理面板显隐、右侧占位、分组选项状态、汇总构建输入和卡片点击产生的 `navigationRequest`。
- `OverviewReportBuilder` 作为纯数据边界，接收项目、当前日期、日历和三个选中周期，计算完成内容及待办提醒卡片。
- 周待办不再依赖当前时间轴条目作为唯一来源，而是从活跃项目上的 `Reminder` 配置提取未完成目标；这样单次提醒触发并从时间轴移除后，仍可作为逾期内容持续展示。
- `NotificationScheduleService` 仅针对项目级单次提醒补齐“触发后保留配置”的行为，使项目、任务和子任务在汇总面板中的逾期可见性一致；保留配置不意味着重复发通知。
- `OverviewReportDrawerView` 只渲染下拉、复制、卡片提醒信息和点击入口，不承担提醒生命周期或筛选业务。

不扩展 `MenuBarContentBuilder`：菜单栏以今日操作和完成按钮为核心，而总览面板以累计待办、历史区间、复制及导航为核心，行为边界不同。

## 面板布局与交互

### 抽屉框架

- 展开宽度沿用备忘录面板的 `360pt`。
- 展开时占用总览网格右侧空间，避免覆盖项目卡片；顶栏搜索区域同样避让面板。
- 背景使用 `ViabarColor.mainPanelMemoBackground`，边界层级和卡片暗色适配参考备忘录面板。
- 顶部仅提供右上角 `sidebar.right` 切换按钮。
- 修复按钮点击时，右栏内部按钮的交互层必须高于覆盖顶部的全局工具栏，或工具栏避让右栏范围；按钮仍位于右栏内部。
- 面板收起后，顶部工具栏显示同样风格的重新展开入口。

### 分组标题与周期下拉

- 顺序固定为 `周待办`、`周完成`、`月完成`。
- 标题为紧凑行高的次级灰色文字。
- 标题右侧紧跟小型胶囊下拉，显示当前周期；胶囊使用低强调面板控件背景、小字号文本及向下箭头。
- 三个胶囊选择仅影响当前总览会话的展示状态，不新增持久化设置字段。
- 复制按钮位于标题行最右侧，仅显示 `doc.on.doc` 图标；hover 时手形指针和蓝色图标反馈，复制成功复用 `已复制` 胶囊反馈。
- 空分组保留禁用的复制按钮占位，并显示对应周期的本地化空状态。

## 日期范围与数据来源

构建器接受 `now` 与 `Calendar`，日期区间均使用半开区间 `start <= timestamp < end`。

### 周完成

- `本周` 使用当前自然周区间。
- `上周` 使用紧邻当前自然周之前的完整自然周区间。
- 从全部项目中读取 `completedAt`，包含归档项目。

### 月完成

- `本月` 使用当前自然月区间。
- `上月` 使用紧邻当前自然月之前的完整自然月区间，并作为默认选择。
- 从全部项目中读取 `completedAt`，包含归档项目。

### 周待办

- `本周` 的截止边界为当前自然周结束时间。
- `下周` 的截止边界为下一自然周结束时间。
- 从所有未归档项目中的提醒配置读取内容：项目 `reminder`、未完成任务 `reminder`、未完成子任务 `reminder`。
- 对每个提醒，仅当其展示触发时间早于所选周的结束边界时纳入，且不要求落在该周开始时间之后。因此更早已逾期而仍未完成的任务会持续出现在后续周视图中。
- 已完成目标和归档项目即使仍保留提醒配置也不显示。
- 循环提醒沿用调度服务推进后的下一次触发时间；单次提醒保留其原触发时间并持续呈现逾期状态。

## 提醒生命周期修正

### 任务与子任务

- 保持现有行为：单次通知触发后删除时间轴投递条目，但保留 `Reminder` 与原触发时间；完成任务、删除提醒或归档项目时按现有路径清理/隐藏。
- 循环提醒触发后继续推进至下一次未来时间并重建时间轴条目。

### 项目级提醒

- 单次项目提醒触发后不再将 `project.reminder` 设为 `nil`。
- 触发后保留原触发时间并记录 `lastTriggeredTimestamp`，供周待办持续以红色显示；不得由该保留配置重复投递通知。
- 后续调用 `syncProject` 时，若单次提醒的 `lastTriggeredTimestamp` 已覆盖其 `fireTimestamp`，不得重新插入对应时间轴条目；用户重新设置提醒时间后才重新获得一次待投递机会。
- 循环项目提醒继续推进下一次未来触发时间并重建时间轴。
- 归档项目或项目已经没有未完成任务时，周待办不显示项目级提醒；现有归档清理时间轴语义保持不变。

## 卡片与任务树规则

### 完成卡片

- 无子任务父任务：自身完成时间位于选中区间内时显示。
- 含子任务父任务：父任务只作上下文行，仅列出完成时间位于选中区间内的子任务。
- 完成卡片不显示完成时间或提醒时间。
- 归档项目完成卡片右上角显示次级灰色 `已归档`。

### 待办卡片聚合

- 每个项目至多生成一张卡片。
- 项目存在符合截止边界的项目提醒时，卡片右上角显示 `alarm.fill` 和项目提醒时间；如项目同时为星标项目，项目提醒位于 `star.fill` 左侧。
- 项目级提醒继续映射项目当前第一条未完成父任务；若该父任务包含未完成子任务，父任务下缩进列出全部未完成子任务。
- 父任务自己的提醒显示该父任务行；子任务自己的提醒显示父任务上下文及命中的缩进子任务。
- 项目提醒展开内容与任务/子任务提醒内容合并，父任务按 `milestoneId`、子任务按 `taskId` 去重。
- 卡片内部任务树按项目现有 `orderIndex` 顺序显示。

### 提醒标识与状态色

- 所有任务和子任务行前显示较小的 `circle.fill`，颜色与该行文本一致。
- 任务或子任务自身有纳入当前待办范围的提醒时，在文本前显示 `alarm.fill` 与提醒时间。
- 项目提醒时间只显示在卡片右上角，不重复加到其映射出的每一条任务行前。
- 提醒对应的图标和文本颜色按相对 `now` 状态显示：
  - 提醒时间已经过去：红色；
  - 提醒时间在今天且尚未到达：橙色；
  - 提醒时间为明天或更远的未来：次级灰色。
- 没有自身提醒、仅因项目提醒展开的任务行使用普通正文颜色。

### 点击导航

- 项目名称可点击，生成 `.project` 导航请求，进入项目详情。
- 父任务行可点击，生成 `.milestone(milestoneId)` 导航请求。
- 子任务行可点击，生成 `.subTask(milestoneID:subTaskID:)` 导航请求。
- 对任务/子任务的跳转使用现有 `MilestoneListView` 与 `SearchTargetHighlight` 的橙色高亮和滚动行为，不新增另一套定位效果。
- 归档完成卡片的项目/任务亦可复用现有归档导航展开行为，不改变项目归档状态。

## 卡片视觉设计

- 卡片圆角、底色、描边与纵向节奏参考菜单栏下拉卡片，面板背景与深浅色适配沿用备忘录面板语义。
- 项目标题左侧 SF Symbol 使用项目主题色；项目名称统一使用 `ViabarColor.primary`。
- 任务树文本允许多行显示，子任务保持明显缩进。
- 卡片不提供复选框或编辑动作。

## 复制规则

- 复制仅包含当前分组、当前胶囊选择下正在展示的项目与任务树。
- 不复制周期标签、提醒时间、提醒图标、状态色、完成时间、`已归档`、星标或 UI 反馈。
- 项目前使用流水序号；父任务/无子任务任务行前为 `- `；子任务行前为两个空格加 `- `；项目块间保留空行。

示例：

```text
1. App 发布
- 审核材料整理
  - 补齐截图与说明文本
  - 检查多语言商店文案

2. 网站改版
- 设计走查安排
```

## 本地化

新界面文本需加入 `Viabar/zh-Hans.lproj/Localizable.strings` 与 `Viabar/en.lproj/Localizable.strings`：

- `周待办`、`周完成`、`月完成`
- `本周`、`下周`、`上周`、`本月`、`上月`
- `已归档`
- 三个分组的复制帮助文本与各周期空状态
- `收起汇总面板`、`展开汇总面板`

复制正文中的项目与任务名称始终保留用户输入原文。

## 代码职责边界

- `Viabar/ContentView.swift`
  - 管理抽屉显隐、三个周期选择、右侧占位、点击后的 `navigationRequest` 以及折叠按钮层级修复。
- `Viabar/Models/OverviewReport.swift`
  - 定义周期选项、只读项目卡片、提醒展示元数据和导航目标；
  - 计算完成区间、累计截止待办、项目/任务提醒合并、去重和复制正文。
- `Viabar/Views/MainPanel/OverviewReportDrawerView.swift`
  - 渲染小型胶囊下拉、复制反馈、提醒状态样式和可点击卡片任务树。
- `Viabar/Services/NotificationScheduleService.swift`
  - 使项目级单次提醒触发后保留其提醒配置，仅停止再次投递。
- `Viabar/Models/ReminderDisplay.swift`
  - 若需要，集中提供按参考日期计算待办提醒颜色状态的纯函数。
- `ViabarTests/ViabarTests.swift`
  - 覆盖构建器、复制、导航目标与项目单次提醒触发后保留行为。

## 验证策略

### 测试源码范围

- 周完成在本周/上周选项下筛选正确，月完成在本月/上月选项下筛选正确，且默认选项为本周/本周/上月。
- 完成内容对含子任务的父任务仅展示选中周期内完成的子任务，并包含归档项目历史。
- 周待办在本周和下周选择下都采用截止周末累计过滤；下周视图包含此前仍未完成的逾期提醒。
- 待办不显示归档项目或已完成目标。
- 项目提醒、任务提醒和子任务提醒合并到单一项目卡片，并按任务身份去重。
- 项目提醒出现在卡片头部，任务/子任务自身提醒携带提醒时间及其导航目标。
- 逾期、今天尚未到期、明天及以后三个提醒状态分类正确。
- 项目级单次提醒触发后保留配置且不会重建投递时间轴；循环提醒仍推进至下一未来时间。
- 项目、父任务、子任务输出正确的 `GlobalSearchDestination`，供现有橙色高亮导航使用。
- 复制正文不包含任何提醒与状态信息。

### 静态与视觉核验

- 核对三组标题、胶囊顺序与默认值，对照已确认视觉稿。
- 核对展开按钮位于面板内部并可命中，收起后重新展开入口可见。
- 核对提醒图标位置、`circle.fill`、三档颜色、多行文本、归档标签与项目蓝色标题。
- 核对新增文本加入两份本地化资源且浅色/深色语义沿用现有风格。
- 默认运行 `git diff --check` 与字符串资源语法检查；除非用户明确授权，不运行编译或测试命令。
