# iOS Static Interaction Prototype Design

## Goal

在现有 iOS 基础骨架上实现一轮可操作的高保真静态交互原型，用于验证移动端页面结构、视觉密度和核心操作手感。

本轮只使用内存中的演示数据。用户可以在运行期间体验临时新增、编辑、删除、勾选和搜索，但 App 重启后恢复初始演示内容。

## Scope

本轮实现三个主要页面：

1. 总览。
2. 项目详情 - 任务。
3. 项目详情 - 备忘录。

同时提供以下辅助入口：

- 总览底部的报告和归档占位页。
- 全局搜索展开态和静态数据搜索结果。
- 项目、任务、子任务和备忘录的原生长按菜单。
- 项目详情中的新增输入框。

本轮不实现：

- SwiftData 持久化读写。
- 真实业务数据的增删改查。
- CloudKit 或 iCloud 同步。
- SwiftData schema 修改。
- 报告和归档的正式内容。
- 设置页正式功能。

## Architecture

### Single In-Memory State Source

新增 `IOSPrototypeStore`，作为 iOS 静态原型唯一的内存状态源。它负责：

- 保存演示项目、里程碑、子任务和备忘录。
- 保存临时收藏、归档和删除结果。
- 保存临时完成状态并计算项目进度。
- 保存页面导航、当前 Tab、搜索展开态和编辑态。
- 在新增、编辑、删除和勾选操作后更新当前运行期间的展示。

原型数据结构独立于 SwiftData `@Model`。本轮不打开真实业务数据写入口，也不为静态原型修改数据库结构。

### View Boundaries

视图拆分为：

- `IOSPrototypeRootView`：根导航与总览底部 Tab。
- `IOSOverviewView`：总览分组和项目卡片列表。
- `IOSOverviewProjectCard`：复用的项目卡片。
- `IOSProjectDetailView`：项目详情容器、标题栏、进度、提醒和底部 Tab。
- `IOSProjectTasksView`：两级任务列表。
- `IOSProjectMemosView`：备忘录列表。
- `IOSPrototypeSearchView`：展开式搜索框与结果列表。
- `IOSMultilineEditor`：统一的多行新增和原地编辑输入组件。
- `IOSPlaceholderView`：报告和归档占位页。

这些组件只依赖 `IOSPrototypeStore` 暴露的内存状态和操作。后续接入 `ProjectService` 时，优先替换数据来源和操作闭包，不重新设计页面结构。

## Navigation

### Overview

App 启动后进入总览。

底部左侧胶囊导航包含：

1. 总览。
2. 报告。
3. 归档。

底部右侧保留独立搜索按钮。点击后展开搜索框，再次点击外部区域可收起。

报告和归档在本轮显示占位页面，保留后续正式接入位置。

### Project Detail

点击总览中的项目卡片进入项目详情。

项目详情顶部包含：

- 返回按钮。
- 项目图标与项目名称。
- 右上角更多菜单。
- 项目进度。
- 项目级提醒摘要。

底部左侧胶囊导航包含：

1. 任务。
2. 备忘录。

底部右侧保留独立新增按钮：

- 任务 Tab 中新增里程碑。
- 备忘录 Tab 中新增备忘录。

## Overview Visual Design

总览页面延续 macOS 当前密度，避免使用占据视觉焦点的大标题。

分组标题：

- “星标项目”和“其他项目”使用小号、低对比度、紧凑间距的 macOS 风格标签。
- 分组标题只表达层级，不承担卡片状态展示。

项目卡片：

- 白色圆角容器。
- 左侧显示项目主题色细线。
- 显示项目名称。
- 显示顶端未完成里程碑。
- 如存在子任务，显示首个未完成子任务。
- 显示项目提醒摘要。
- 显示进度百分比和进度环。
- 星标项目卡片右上角显示 `star.fill`，与 macOS `OverviewProjectCard` 保持一致。

星标项目与其他项目复用同一张卡片组件，仅按分组展示。

## Task Detail Design

任务页使用两级结构：

```text
Milestone
└── SubTask
```

不可增加第三层任务。

行内展示：

- 左侧完成状态圆圈。
- 中部任务标题。
- 可选提醒摘要。
- 右侧提醒图标。
- 子任务增加缩进，保持层级可读。

点击完成状态圆圈后，只在内存中切换完成状态，并立即更新项目进度。

点击任务标题或子任务标题后，当前文字原地切换为多行编辑器，不跳转页面。

## Memo Detail Design

备忘录页显示紧凑卡片列表：

- 时间。
- 多行备忘录内容。

点击备忘录后，当前内容原地切换为多行编辑器，不跳转页面。

## Multiline Input Interaction

新增输入框和原地编辑器复用 `IOSMultilineEditor`。

输入规则：

- `Return` 输入换行。
- 键盘工具栏中的“完成”按钮提交。
- 点击编辑器外部区域提交。
- 空白新增内容不创建临时条目。
- 空白编辑内容不提交覆盖，保留原内容。

项目详情右侧新增按钮负责展开新增输入框。点击外部区域时，提交有效内容并关闭输入框。

## Context Menus

长按使用 iOS 原生 `.contextMenu`。菜单语义对齐 macOS 右键菜单入口。

项目卡片菜单：

- 编辑。
- 归档。
- 收藏或取消收藏。
- 删除。

任务和子任务菜单：

- 编辑。
- 删除。

备忘录菜单：

- 编辑。
- 删除。

本轮菜单操作只修改 `IOSPrototypeStore` 内存状态。

## Search

总览右侧搜索按钮展开搜索框，搜索范围覆盖：

- 项目名称。
- 里程碑名称。
- 子任务名称。
- 备忘录内容。

点击结果后：

- 项目结果进入对应项目详情。
- 任务和子任务结果进入对应项目详情的任务 Tab。
- 备忘录结果进入对应项目详情的备忘录 Tab。

本轮不实现搜索目标高亮动画，但保留后续复用 macOS `SearchTargetHighlight` 语义的位置。

## Compatibility And Data Safety

- 最低系统版本保持 iOS 17.6。
- 使用 SwiftUI 和 iOS 17 可用 API。
- 不使用 iOS 26 专属 Liquid Glass API。
- 不修改 macOS 页面行为。
- 不修改 SwiftData schema。
- 不写入真实业务数据库。
- 不启用 CloudKit。
- 不删除或迁移任何数据库文件。

## Static Verification

实现后增加静态检查，至少覆盖：

- iOS 原型视图文件存在。
- `IOSPrototypeStore` 使用独立内存演示数据。
- iOS 原型不引入新的 `@Model`。
- iOS 原型不直接调用 SwiftData 写入口。
- 长按菜单、搜索展开、详情 Tab 和多行编辑器入口存在。
- `git diff --check` 通过。
- 相关 plist 使用 `plutil -lint` 检查。

## Manual Verification

由用户在 Xcode 中手动运行 iOS App 后验证：

1. App 打开后显示总览。
2. “星标项目”和“其他项目”使用紧凑小号标签。
3. 星标项目卡片右上角显示 `star.fill`。
4. 点击项目卡片进入详情。
5. 任务和备忘录 Tab 可以切换。
6. 点击任务、子任务和备忘录后可以原地编辑多行文本。
7. `Return` 可以换行。
8. 点击键盘工具栏“完成”或外部区域可以提交临时内容。
9. 点击新增按钮后可以新增临时里程碑或备忘录。
10. 长按项目、任务、子任务和备忘录可以打开原生功能菜单。
11. 勾选任务后项目进度立即变化。
12. 点击搜索按钮后可以展开搜索框，并进入对应项目详情。
13. 报告和归档入口可以进入占位页。
14. 重启 App 后恢复初始演示数据。
