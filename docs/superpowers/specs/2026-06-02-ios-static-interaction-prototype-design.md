# iOS Static Interaction Prototype Design

## Goal

在现有 iOS 基础骨架上实现一轮可操作的高保真静态交互原型，用于验证移动端页面结构、视觉密度和核心操作手感。

本轮只使用内存中的演示数据。用户可以在运行期间体验临时新增、编辑、删除、勾选和搜索，但 App 重启后恢复初始演示内容。

## Scope

本轮实现三个主要页面：

1. 总览。
2. 项目详情 - 任务。
3. 项目详情 - 备忘录。

同时提供以下辅助入口：

- 总览底部的报告占位页和归档树页面。
- 全局搜索展开态和静态数据搜索结果。
- 项目、任务、子任务和备忘录的原生长按菜单。
- 项目详情中的新增输入框。
- 总览右上角回收站占位入口。

本轮不实现：

- SwiftData 持久化读写。
- 真实业务数据的增删改查。
- CloudKit 或 iCloud 同步。
- SwiftData schema 修改。
- 报告的正式内容。
- 回收站正式内容。
- 设置页正式功能。

## Architecture

### Single In-Memory State Source

新增 `IOSPrototypeStore`，作为 iOS 静态原型唯一的内存状态源。它负责：

- 保存演示项目、里程碑、子任务和备忘录。
- 保存演示归档文件夹树。
- 保存临时收藏、归档和删除结果。
- 保存临时完成状态并计算项目进度。
- 保存页面导航、当前 Tab、搜索展开态和编辑态。
- 在新增、编辑、删除和勾选操作后更新当前运行期间的展示。

原型数据结构独立于 SwiftData `@Model`。本轮不打开真实业务数据写入口，也不为静态原型修改数据库结构。

### View Boundaries

视图拆分为：

- `IOSPrototypeRootView`：根导航与总览底部 Tab。
- `IOSOverviewView`：总览分组和项目卡片列表。
- `IOSOverviewProjectCard`：复用的项目卡片。
- `IOSProjectDetailView`：项目详情容器、标题栏、进度、提醒和底部 Tab。
- `IOSProjectTasksView`：两级任务列表。
- `IOSProjectMemosView`：备忘录列表。
- `IOSPrototypeSearchView`：展开式搜索框与结果列表。
- `IOSPrototypeDetailComposer`：总览和项目详情共用的底部多行输入框，用于新增和编辑。
- `IOSPrototypeArchiveView`：归档文件夹树与归档项目胶囊列表。
- `IOSPrototypeArchiveFolderNodeView`：按展开状态递归创建子树的归档文件夹节点。
- `IOSPrototypeArchivedProjectRow`：紧凑归档项目行。
- `IOSPlaceholderView`：报告和回收站占位页。

这些组件只依赖 `IOSPrototypeStore` 暴露的内存状态和操作。后续接入 `ProjectService` 时，优先替换数据来源和操作闭包，不重新设计页面结构。

## Navigation

### Overview

App 启动后进入总览。

底部左侧胶囊导航包含：

1. 总览。
2. 报告。
3. 归档。

底部右侧保留独立搜索按钮。点击后展开搜索框，右侧按钮切换为关闭图标，再次点击关闭搜索。

报告在本轮显示占位页面。归档页实现静态文件夹树和临时内存交互。

底部 Tab 使用小号规格：

- 胶囊与独立按钮高度约 `50pt`。
- 图标约 `16pt`。
- 文案使用 `caption2`。
- 保留足够点击区域，同时减少视觉占用。

总览右上角增加 `trash` 图标。本轮只进入回收站占位页，后续再接入现有独立 `trash.store`。

### Project Detail

点击总览中的项目卡片进入项目详情。

项目详情顶部包含：

- 返回按钮。
- 项目图标与项目名称。
- 右上角更多菜单。
- 项目进度。
- 项目级提醒摘要。

底部左侧胶囊导航包含：

1. 任务。
2. 备忘录。

底部右侧保留独立新增按钮：

- 任务 Tab 中新增里程碑。
- 备忘录 Tab 中新增备忘录。
- 长按里程碑后选择“新增子任务”，新增到对应里程碑。底部 `+` 不承担新增子任务入口，因为它没有父里程碑上下文。

## Overview Visual Design

总览页面延续 macOS 当前密度，避免使用占据视觉焦点的大标题。

分组标题：

- “星标项目”和“其他项目”使用小号、低对比度、紧凑间距的 macOS 风格标签。
- 分组标题只表达层级，不承担卡片状态展示。

项目卡片：

- 白色圆角容器。
- 左侧显示完整高度的 `4pt` 主题色直线，不增加上下内缩。
- 完成项目的左边框和项目 SF Symbol 使用 `ViabarColor.success`；未完成项目使用项目主题色。
- 显示项目名称。
- 顶端未完成里程碑使用 `mappin.and.ellipse` 图标，字体、灰度和缩进与 macOS `OverviewProjectCard` 一致。
- 如存在子任务，首个未完成子任务使用 `list.bullet.indent` 图标，采用 macOS 相同的额外缩进和文本灰度。
- 显示顶端未完成里程碑的提醒摘要。
- 进度环固定使用 `#00BBE1 -> #00F9D0 -> #00BBE1` 的环形渐变，轨道为 `#00BBE1` 的低透明度版本。
- 进度百分比固定使用 `#00BBE1`，不跟随项目主题色。
- 进度百分比强制单行显示，并为 `100%` 保留足够宽度。
- 星标项目卡片右上角显示 `star.fill`，固定使用 `ViabarColor.warning`，与 macOS `OverviewProjectCard` 保持一致。

星标项目与其他项目复用同一张卡片组件，仅按分组展示。

## Task Detail Design

任务页使用两级结构：

```text
Milestone
└── SubTask
```

不可增加第三层任务。

行内展示：

- 左侧完成状态圆圈。
- 中部任务标题。
- 可选提醒摘要。
- 右侧提醒图标。
- 子任务增加缩进，保持层级可读。

点击完成状态圆圈后，只在内存中切换完成状态，并立即更新项目进度。

点击任务标题或子任务标题后，在详情底部输入框中加载原文本，不在列表原位置切换编辑器。

长按里程碑弹出的菜单顶部增加“新增子任务”。选择后进入详情底部新增模式，并记录父里程碑。

## Memo Detail Design

备忘录页显示紧凑卡片列表：

- 时间。
- 多行备忘录内容。

点击备忘录后，在详情底部输入框中加载原文本，不在卡片原位置切换编辑器。

## Input And Editing Interaction

输入规则：

- `Return` 只输入换行。
- 不提供键盘工具栏“完成”按钮。
- 不提供其他额外“完成”按钮。
- 保存出口只有右下角 `paperplane.fill`。
- 点击外部区域不保存、不退出编辑模式，只允许系统按正常行为收起键盘。

### Add Mode

- 点击右下角 `+` 后，底部输入框向左展开并自动聚焦，键盘立即弹出。
- 任务 Tab 新增里程碑；备忘录 Tab 新增备忘录。
- 长按里程碑选择“新增子任务”后，同样展开底部输入框，并保存父里程碑 ID。
- 新增模式中隐藏任务 / 备忘录 Tab。
- 右下角 `+` 切换为 `paperplane.fill`。
- 点击纸飞机提交并关闭。
- 空白文本不创建条目。

### Edit Mode

- 总览项目卡片的长按菜单选择“编辑”后，在总览底部输入框中加载项目名称。
- 点击任务、子任务或备忘录后，在详情底部输入框中加载原文本。
- 长按任务、子任务或备忘录菜单选择“编辑”后，也进入同一个底部输入框。
- 列表行和备忘录卡片始终保持普通展示状态，不允许因为编辑会话切换为行内输入框。
- 编辑模式中底部显示与新增模式相同的多行输入框。
- 编辑模式中隐藏任务 / 备忘录 Tab。
- 右下角 `+` 切换为 `paperplane.fill`。
- 点击纸飞机提交并关闭。
- 空白文本删除当前条目。
- 点击外部区域不保存、不退出编辑模式。

## Context Menus

长按使用 iOS 原生 `.contextMenu`。菜单语义对齐 macOS 右键菜单入口。

项目卡片菜单：

- 编辑。
- 归档。
- 收藏或取消收藏。
- 删除。

任务和子任务菜单：

- 新增子任务，仅里程碑菜单显示。
- 编辑。
- 复制。
- 删除。

备忘录菜单：

- 编辑。
- 复制。
- 删除。

本轮菜单操作只修改 `IOSPrototypeStore` 内存状态。

任务和子任务无论是否已完成，都必须保留长按菜单。列表行为不能因为完成状态、删除线样式或完成按钮的命中区域而失效。

## Search

总览右侧搜索按钮展开搜索框，搜索范围覆盖：

- 项目名称。
- 里程碑名称。
- 子任务名称。
- 备忘录内容。
- 活跃项目和归档项目。

展开搜索时：

- 隐藏总览 / 报告 / 归档底部 Tab。
- 搜索框从右侧向左展开，并保持在最底部。
- 结果列表从搜索框上方向上展开。
- 结果行参考 macOS `GlobalSearchOverlay`：使用小型 SF Symbol、标题和路径文本，不为图标增加额外圆角矩形。

搜索结果保存目标实体 ID。点击后：

- 项目结果进入对应项目详情。
- 任务和子任务结果进入任务 Tab，滚动到目标行并显示约 `5` 秒橙色整行背景。
- 备忘录结果进入备忘录 Tab，滚动到目标卡片并显示约 `5` 秒橙色闪烁描边。
- 归档项目结果先进入归档页，展开目标文件夹及完整祖先链，再进入对应归档项目详情；不改变归档状态。

## Favorite And Deletion

- 总览卡片和项目详情必须按项目 ID 从 `IOSPrototypeStore` 读取最新项目状态，不能持有修改前的值继续渲染。
- 收藏和取消收藏后，项目立即切换“星标项目 / 其他项目”分组，卡片星标同步显示或移除。
- 星标项目详情顶部名称后显示黄色 `star.fill`。
- 项目详情右上角不提供删除项目入口。
- 删除项目只允许从总览卡片和归档项目长按菜单触发。
- 项目删除采用 macOS 同款两次确认：第一次显示项目名称、任务数和备忘录数；第二次确认永久删除。

## Detail Tap Areas

- 点击里程碑或子任务行时，除右侧闹钟区域外，整行进入底部编辑模式。
- 左侧完成状态按钮仍只切换完成状态，不触发编辑。
- 点击备忘录卡片任意位置进入底部编辑模式。
- 长按菜单行为保持可用。

## Archive

归档页不提供筛选胶囊，也不提供独立搜索框。搜索统一复用全局搜索入口。

### Lazy Folder Tree

- 使用 `LazyVStack` 渲染根目录。
- 每个文件夹节点只在展开时递归创建子文件夹和归档项目视图，避免一次性创建完整树。
- 文件夹行显示展开箭头、文件夹图标、名称和直接子项目数量。
- 点击文件夹行折叠或展开。
- 子层级每级缩进 `16pt`。
- 搜索命中归档项目时自动展开完整祖先链。

### Archived Project Row

- 使用紧凑胶囊横条。
- 显示项目色 SF Symbol、项目名称和小型渐变进度环。
- 不显示百分比、左侧主题色竖线、任务、子任务和提醒。
- 点击进入项目详情。
- 长按菜单包含“取消归档”和“删除项目”。

### Folder Actions

- 右下角独立 `folder.badge.plus` 按钮新增根文件夹。
- 文件夹长按菜单包含“新建子文件夹”、“重命名”和“删除文件夹”。
- 新建根文件夹、新建子文件夹和重命名均通过底部输入框提交。
- 删除空文件夹直接执行。
- 删除包含子文件夹或归档项目的文件夹时弹出二次提示。

## Compatibility And Data Safety

- 最低系统版本保持 iOS 17.6。
- 使用 SwiftUI 和 iOS 17 可用 API。
- 不使用 iOS 26 专属 Liquid Glass API。
- 不修改 macOS 页面行为。
- 不修改 SwiftData schema。
- 不写入真实业务数据库。
- 不启用 CloudKit。
- 不删除或迁移任何数据库文件。
- 归档树和回收站入口本轮只使用内存演示数据，不接入真实 `trash.store`。

## Static Verification

实现后增加静态检查，至少覆盖：

- iOS 原型视图文件存在。
- `IOSPrototypeStore` 使用独立内存演示数据。
- iOS 原型不引入新的 `@Model`。
- iOS 原型不直接调用 SwiftData 写入口。
- 长按菜单、搜索展开、详情 Tab 和多行编辑器入口存在。
- 搜索结果携带目标实体 ID，并具备滚动定位和高亮入口。
- 归档页使用懒加载递归树，并具备根文件夹新增、子文件夹新增、重命名和非空删除确认入口。
- macOS 同款进度环颜色、任务图标、子任务图标、星标色和底部纸飞机入口存在。
- 原型中不存在键盘工具栏“完成”按钮。
- `git diff --check` 通过。
- 相关 plist 使用 `plutil -lint` 检查。

## Manual Verification

由用户在 Xcode 中手动运行 iOS App 后验证：

1. App 打开后显示总览。
2. “星标项目”和“其他项目”使用紧凑小号标签。
3. 星标项目卡片右上角显示 `star.fill`。
4. 点击项目卡片进入详情。
5. 任务和备忘录 Tab 可以切换。
6. 点击任务、子任务和备忘录后，底部输入框加载原文本并自动聚焦。
7. `Return` 可以换行，键盘上方不显示额外“完成”按钮。
8. 点击新增按钮后，底部输入框向左展开并自动聚焦；右下角按钮切换为纸飞机。
9. 新增或编辑过程中，任务 / 备忘录 Tab 隐藏。
10. 长按项目并选择“编辑”后，总览底部输入框加载项目名称。
11. 新增和编辑都只能点击纸飞机提交；点击外部区域不会保存或退出。
12. 新增空白内容不创建条目；编辑为空时删除当前条目。
13. 长按项目、任务、子任务和备忘录可以打开原生功能菜单。
14. 长按任务、子任务和备忘录后可以复制其文本。
15. 已完成的任务和子任务仍然可以打开长按菜单。
16. 长按里程碑后可以通过菜单新增子任务，并在底部输入框提交。
17. 勾选任务后项目进度立即变化。
18. 点击搜索按钮后，底部 Tab 隐藏，搜索框向左展开，结果列表向上展开。
19. 点击任务、子任务和备忘录搜索结果后，进入对应详情位置并显示橙色高亮。
20. 归档页显示可折叠的多级文件夹树，项目使用紧凑胶囊横条。
21. 右下角可以新增根文件夹；长按文件夹可以新增子文件夹、重命名和删除。
22. 删除非空文件夹时出现二次提示。
23. 删除项目时出现两次确认，详情页右上角没有删除入口。
24. 收藏和取消收藏后，分组、卡片星标和详情标题星标同步变化。
25. 总览右上角回收站图标进入占位页。
26. 报告入口可以进入占位页。
27. 重启 App 后恢复初始演示数据。
