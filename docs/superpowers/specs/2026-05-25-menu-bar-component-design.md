# Menu Bar Component Design

日期：2026-05-25

## 目标

为 Viabar 增加一个可配置的 macOS 菜单栏组件，以轻量卡片面板呈现当前需要行动的任务，并允许直接完成任务或快速添加新任务。组件与主应用共享 SwiftData 数据、提醒调度和跳转能力；面板内的高频筛选配置与设置窗口双向同步。

本设计延续现有能力：

- `AppSettings` 作为唯一设置记录；
- `ProjectService` 负责任务创建与完成状态更新；
- `NotificationScheduleService` 负责提醒时间轴同步；
- `GlobalSearchNavigationRequest` 负责项目、任务与子任务的定位及橙色强调反馈；
- 应用保存的主题选择负责主窗口与菜单栏面板的外观一致性。

## 已确认的产品决策

- 使用原生 SwiftUI `MenuBarExtra`，不引入独立辅助应用或第二套存储。
- 菜单栏组件只展示未归档项目；“星标项目”是在未归档项目上继续过滤。
- 菜单栏项目范围独立于主窗口总览的项目范围设置。
- 面板右下角齿轮菜单只提供项目范围与内容模式的即时切换，并写回同一份设置记录。
- “当前任务”模式下，每个项目至多呈现一条当前内容。
- “提醒任务”模式下，一个项目可以呈现多个到期或当天待触发的提醒内容，并按提醒时间先后显示。
- 项目级提醒映射到项目当前第一条未完成任务或子任务，同时保留“项目提醒”的来源标识。
- 任务与子任务的循环提醒触发后推进到下一次未来触发时间；单次提醒触发后保留逾期提醒配置，直到用户处理或修改。
- 底部快速添加仅创建所选项目下的新任务，不创建子任务；提醒配置为可选。
- 本轮实施与排查遵守仓库要求：未另行授权时不通过编译代码排查问题。

## 实现方案

采用 `MenuBarExtra` 与主应用共用模型容器和服务层的方案：

- `ViabarApp` 提供主窗口、设置窗口与菜单栏场景；
- `MenuBarExtra` 的插入状态绑定 `menuBarComponentEnabled`，因此设置中的启用开关可即时显示或移除状态项；
- 三个场景读取相同的 `AppSettings` 和项目模型；
- 菜单栏的写操作调用现有服务，而不在视图内另建任务更新规则；
- 菜单栏点击跳转通过应用级协调器向现有主窗口投递导航请求；
- 菜单栏面板按照保存的主题值渲染，使用 SwiftUI/AppKit 现有外观应用边界。

此方案避免了 AppKit `NSStatusItem` / `NSPopover` 的额外生命周期管理，也避免了独立进程带来的数据通信与主窗口唤起复杂度。

## 设置与持久化

### 设置窗口布局

在“设置 - 通用”中，将现有“启用菜单栏组件”从“启动”组移出，新增独立的“菜单栏组件”组，内容依次为：

1. `启用`
2. `菜单栏图标`
3. `项目`
4. `功能`

“启动”组仅保留开机启动相关设置。

### 菜单栏图标选项

设置页允许选择以下 SF Symbol，按此顺序展示：

1. `bookmark`
2. `bookmark.fill`
3. `bookmark.circle`
4. `bookmark.circle.fill`
5. `star.rectangle`
6. `star.rectangle.fill`
7. `list.bullet.rectangle`
8. `list.bullet.rectangle.fill`
9. `checkmark.seal`
10. `checkmark.seal.fill`
11. `checkmark.rectangle`
12. `checkmark.rectangle.fill`

菜单栏状态项使用当前保存的符号。无效存储值回退到 `bookmark.fill`。

### 新增设置字段

`AppSettings` 保留 `menuBarComponentEnabled` 并新增：

- `menuBarIcon: String = "bookmark.fill"`
- `menuBarProjectScope: String = "allProjects"`
- `menuBarContentMode: String = "currentTask"`

`menuBarProjectScope` 支持：

- `allProjects`：全部未归档项目；
- `favoriteProjects`：仅未归档且 `isFavorite == true` 的项目。

`menuBarContentMode` 支持：

- `currentTask`：当前任务；
- `reminderTask`：提醒任务。

无效存储值分别回退为 `allProjects` 与 `currentTask`。

菜单栏使用独立的 `menuBarProjectScope`，不读取或覆盖主窗口总览使用的 `overviewScope`。

### 面板内配置菜单

面板右下角提供齿轮配置按钮，点击后打开轻量菜单，包含：

- `项目` 子菜单：`全部项目` / `星标项目`；
- `功能` 子菜单：`当前任务` / `提醒任务`。

菜单选项与设置窗口读写相同字段，选择后立即刷新面板内容，并同步反映到已打开的设置窗口。`启用` 和 `菜单栏图标` 仅由完整设置窗口管理，不放入面板配置菜单。

## 菜单栏面板外观

面板参考提供的浅色圆角卡片风格：

- 使用具有圆角和留白的纵向面板布局；
- 项目以分组卡片呈现，每个卡片包含项目名称与可操作任务行；
- 任务行前方使用可点击复选框；
- 设置过提醒的条目在任务文本下方展示较小字号的下次提醒时间与循环周期；
- 面板底部放置单行快速添加区域；
- 右下角放置齿轮配置按钮。

面板使用 `AppSettings.theme` 的当前选择：

- `light` 强制浅色呈现；
- `dark` 强制深色呈现；
- `system` 跟随系统外观。

具体颜色与阴影实现应沿用应用现有颜色及可读性规则，而不复制参考图中的品牌色。

## 项目范围与卡片排序

菜单栏只处理活跃项目：

- 始终排除 `isArchived == true` 的项目；
- 在 `favoriteProjects` 模式下进一步排除未星标项目；
- 可展示项目沿用现有项目顺序，首先按 `orderIndex` 排列，并在需要稳定次序时使用项目名称作为次级比较条件。

只有存在当前模式可显示内容的项目才创建卡片。当前模式下没有任何可显示项目时，面板呈现简洁空状态，底部快速添加仍保持可用。

## 当前任务模式

`当前任务` 模式为每个项目提取至多一个当前执行目标：

1. 按 `orderIndex` 寻找第一条未完成任务（`Milestone`）。
2. 若该任务没有子任务，显示任务本身。
3. 若该任务存在未完成子任务，显示按 `orderIndex` 排列的第一条未完成子任务。
4. 若该任务有子任务但已经没有未完成子任务，则回退显示任务本身，与现有数据语义保持一致。
5. 子任务行以次级信息标示所属父任务，便于识别层级。
6. 若项目没有未完成任务或子任务，不显示该项目卡片。

该规则与现有 `Project.topUnfinishedTitle` 的语义一致，但菜单栏条目还需保存实际的任务或子任务标识，以支持复选框完成及点击定位。

## 提醒任务模式

### 可见范围

`提醒任务` 模式展示所有符合以下条件的任务条目：

- 对应项目未归档，并符合当前项目范围；
- 实际任务或子任务尚未完成；
- 提醒的下次触发时间不晚于当前自然日的 `23:59:59`。

这同时包含：

- 此前日期已经过期但仍未处理的提醒；
- 今天已经过期的提醒；
- 今天稍后才会触发的提醒。

晚于今天结束时间的未来提醒不显示。

菜单栏的提醒可见性由 `Project`、`Milestone` 与 `SubTask` 上保存的 `Reminder` 配置计算，而不是仅从 `NotificationScheduleEntry` 读取。时间轴记录表示仍需投递的系统通知；单次通知投递完成后可以不再留在时间轴中，但其任务上的逾期提醒配置仍需要继续显示在菜单栏，直至任务完成、提醒被删除或提醒时间被重新设置。

### 任务与子任务提醒

任务（`Milestone`）和子任务（`SubTask`）自身设置的提醒分别生成独立条目。条目展示：

- 任务或子任务标题；
- 子任务对应的父任务提示；
- 下次提醒时间；
- 若为循环提醒，则显示循环周期。

### 项目级提醒

项目自身设置的提醒符合可见范围时，生成一条映射条目：

- 目标内容取该项目当前第一条未完成任务或子任务；
- 行内标示该提醒来源为“项目提醒”；
- 点击内容定位到映射的实际任务或子任务；
- 点击复选框完成映射的实际任务或子任务；
- 完成后按照现有项目提醒语义重新解析下一条未完成目标；若已无未完成内容，项目提醒不再形成可见映射行。

若一个实际任务同时拥有任务级提醒并被项目级提醒映射，则保留两条独立提醒行，因为两条行对应两个真实提醒来源。

### 排序

同一项目卡片中的提醒条目按各自下次提醒触发时间从早到晚排列。触发时间相同时，使用稳定的条目类型和内容顺序作为次级排序，确保面板刷新时行顺序不跳动。

### 任务与子任务的提醒生命周期

现有任务与子任务的提醒调度需要补齐与菜单栏展示一致的生命周期：

- 单次提醒触发后，系统通知只投递一次；`Reminder` 保留原触发时间，使未完成任务继续显示为逾期条目。
- 循环提醒触发后，系统通知投递一次，并将对应 `Reminder.fireTimestamp` 推进到严格晚于当前时间的下一次循环触发时间。
- 循环提醒推进后，为该任务或子任务写入新的 `NotificationScheduleEntry`，从而继续投递下一次系统通知并让菜单栏显示真实的下次提醒时间。
- 若应用在关闭期间错过多个循环周期，重启处理过期条目时直接推进至下一次未来触发时间，不逐次回放历史周期。
- 完成任务、删除提醒、归档项目或修改提醒时间时，继续由既有同步/移除路径清理或重建对应时间轴条目。

项目级提醒继续使用已经建立的同类“推进到下一次未来时间”语义；本功能将相同原则补齐到任务与子任务提醒。

## 卡片操作与数据同步

### 完成任务

每个显示条目前提供复选框：

- 任务条目调用现有任务完成逻辑；
- 子任务条目调用现有子任务完成逻辑；
- 项目提醒映射条目调用其实际映射目标的完成逻辑。

完成操作必须继续经过 `ProjectService`，从而维持现有父子完成关系、项目状态与持久化写入。与提醒相关的任务变化继续通过已有同步钩子更新 `NotificationScheduleService` 的时间轴。SwiftData 更新后，菜单栏内容与主窗口读取相同模型数据并刷新为最新状态。

### 点击导航

项目名称、任务名称与子任务名称均可点击：

- 点击项目名称：打开或唤起主窗口并显示该项目详情；
- 点击任务：打开对应项目，滚动到该任务，并显示已有搜索跳转风格的橙色背景强调；
- 点击子任务：打开对应项目，滚动到该子任务，并显示同样的橙色背景强调；
- 点击项目提醒映射条目：定位到被映射的实际任务或子任务。

导航复用 `GlobalSearchNavigationRequest` 目标模型和 `ContentView` 已有的详情路由/高亮行为。新增应用级导航桥接路径，允许菜单栏场景先唤起主窗口，再将一次性导航请求交给 `ContentView` 消费。主窗口被隐藏或关闭时，应沿用现有运行时控制器展示主窗口的行为，而不复制新的窗口管理规则。

## 快速添加

面板底部提供单行输入框。

### 展开行为

- 输入为空时仅显示输入框；
- 输入任意非空文本后，在输入区中展开 `项目` 与 `提醒时间` 两个辅助选择；
- `项目` 为必选；
- `提醒时间` 为可选。

### 创建规则

- 提交后始终在所选项目下创建一条新任务（`Milestone`）；
- 菜单栏不支持直接创建子任务；
- 可选项目为全部未归档项目，不受面板当前 `全部项目 / 星标项目` 筛选限制；
- 未选择提醒时间时创建普通任务；
- 选择提醒时间时，为新任务创建对应提醒设置，并沿用时间轴同步流程。

### 提交状态

- 文本经去除首尾空白后为空时不可提交；
- 没有任何未归档项目时，项目选择不可用且不能提交；
- 未选择项目时不能提交；
- 创建成功后清空输入文本、所选项目与提醒配置；
- 数据保存后，当前模式的面板列表立即以最新状态重新计算。

## 代码职责边界

- `Viabar/Models/AppSettings.swift`
  - 定义菜单栏图标、项目范围和内容模式选项；
  - 增加持久化字段、默认值和非法值回退逻辑。
- `Viabar/Views/Settings/SettingsView.swift`
  - 将菜单栏启用项迁入独立组；
  - 提供图标、项目范围和功能模式选择控件。
- `Viabar/ViabarApp.swift`
  - 注册以启用设置绑定插入状态的 `MenuBarExtra`；
  - 为面板注入共享模型容器、服务与运行时协调对象；
  - 根据启用设置控制菜单栏场景呈现路径。
- 新的菜单栏面板视图
  - 渲染卡片、任务行、空状态、快速添加区与齿轮菜单；
  - 仅发出操作意图，不自行复制服务层规则。
- 新的菜单栏快照/查询构建逻辑
  - 将项目、提醒和设置转换为稳定的面板卡片及条目；
  - 集中处理当前目标提取、提醒映射、筛选与排序。
- `Viabar/Services/ProjectService.swift`
  - 继续承载新增任务和完成任务/子任务的写操作。
- `Viabar/Services/NotificationScheduleService.swift`
  - 继续承载提醒时间轴写入与重新同步；
  - 补齐任务和子任务循环提醒的未来时间推进及新时间轴条目写入；
  - 保持单次任务/子任务提醒配置在投递后仍可作为逾期内容展示；
  - 菜单栏不另建通知调度机制。
- `Viabar/Models/GlobalSearch.swift` 与应用级运行时/导航协调对象
  - 复用现有导航目标类型；
  - 承担来自菜单栏场景的主窗口唤起和定位请求转发。

## 边界状态与回退行为

- 菜单栏组件关闭时不呈现状态项。
- 已保存图标、项目范围或内容模式不可识别时，采用规定默认值展示并继续正常读取数据。
- 当前模式无内容时显示空状态，不隐藏快速添加区域。
- 星标筛选切换后若无条目，空状态立即出现。
- 已归档项目不因已有提醒而出现在面板内，也不能成为快速添加目标。
- 项目级提醒没有可映射的未完成任务或子任务时不显示。
- 操作导致提醒条目或当前目标失效时，面板从共享模型中重新计算，不保留过期的 UI 快照。
- 单次系统通知已经投递但任务仍未完成时，提醒任务模式继续展示该逾期任务，不因时间轴记录已消费而消失。

## 验证策略

在实现阶段，为集中且纯粹的菜单栏条目构建逻辑添加测试覆盖：

- 当前任务模式每项目只返回一条；
- 有未完成子任务时优先返回第一条子任务；
- 提醒任务模式包含过期和今天尚未触发的条目，排除明日及以后条目；
- 同一项目的多条提醒按触发时间升序排列；
- 项目级提醒映射到当前实际目标并保留来源标识；
- 单次任务/子任务提醒投递后仍作为逾期配置可见；
- 循环任务/子任务提醒推进至下一次未来触发时间并重建时间轴记录；
- 归档项目排除与星标过滤正确；
- 无效设置值回退正确。

通过代码检查确认：

- 设置页与齿轮菜单写入同一 `AppSettings` 字段；
- 菜单栏写操作调用现有 `ProjectService` 与提醒同步路径；
- 菜单栏点击导航复用现有任务/子任务橙色定位反馈；
- 主题选择应用到菜单栏面板呈现。

按项目约束，除非用户明确授权，实施与排查不运行编译或构建命令。静态验证至少运行 `git diff --check`；测试或构建只有在获得授权后执行。
